# FAQ

## Q1. 프론트엔드에서 전부 처리해도 되나요?

권장하지 않습니다. 서명 검증, 정산 검증, 멱등성 저장은 서버에서 처리하는 것이 안전합니다.

## Q2. legacy 헤더만 들어오는데 괜찮나요?

괜찮습니다. `adaptLegacyToV2Canonical`로 내부 포맷을 통일해서 처리하면 됩니다.

## Q3. TTM 해시는 꼭 써야 하나요?

실무에서는 사실상 필수에 가깝습니다. 사용자가 승인한 내용과 실제 정산 데이터를 강하게 묶어줍니다.

## Q4. fail-closed는 언제 켜야 하나요?

법적/보안 요구가 높은 서비스(금융/정산 핵심 경로)에서는 기본 활성화를 권장합니다.

## Q5. replay store는 어떤 저장소를 써야 하나요?

개발/테스트는 메모리 스토어, 운영은 Redis/DB 같은 외부 저장소를 권장합니다.

## Q6. 이 SDK가 WebAuthn 자체를 처리하나요?

SDK는 검증기 인터페이스(`AssertionVerifier`)를 제공합니다. 실제 WebAuthn 검증 로직은 서비스에서 구현해야 합니다.

## Q7. 체인 finality 정책은 어디서 관리하나요?

`executeSettlement` 옵션의 `finality.hook` 구현체에서 관리합니다. 체인별 정책을 분리해 운영하세요.
